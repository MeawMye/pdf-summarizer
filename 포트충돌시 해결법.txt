포트 충돌의 원인을 바로 제거하기 위해 3001 점유 프로세스를 찾아 종료하고 서버 재기동까지 확인하겠습니다.

Ran terminal command: Get-NetTCPConnection -LocalPort 3001 -ErrorAction SilentlyContinue | Format-Table -AutoSize; $conn = Get-NetTCPConnection -LocalPort 3001 -ErrorAction SilentlyContinue; if ($conn) { Get-Process -Id $conn.OwningProcess | Format-List Id,ProcessName,Path; Stop-Process -Id $conn.OwningProcess -Force; Write-Output "Stopped PID $($conn.OwningProcess)" } else { Write-Output "Port 3001 is free" }

충돌 프로세스 종료가 완료되었습니다. 같은 포트로 정상 기동되는지 바로 검증하겠습니다.

Ran terminal command: npm start

에러 원인 해결됐습니다. 3001 포트를 점유하던 기존 Node 프로세스(PID 22316)를 종료했고, 이후 서버가 정상 기동됐습니다.

현재 상태
- Node API 서버 정상 실행: http://localhost:3001

왜 발생했는지
- 이전에 실행 중이던 서버가 남아 있어서 같은 포트 재사용 충돌(EADDRINUSE)이 났습니다.

다음부터 같은 에러가 나면 바로 이렇게 처리하면 됩니다
1. 포트 점유 확인
   Get-NetTCPConnection -LocalPort 3001
2. 점유 프로세스 종료
   Stop-Process -Id PID번호 -Force
3. 서버 재시작
   npm start
